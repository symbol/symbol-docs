クライアント

Symbol ベースのネットワークは、トラストレスな、高性能で安全なブロックチェーンプラットフォームを提供するためにノードに依存しています。

これらのノードは2015年から稼働し続けてきた、従来 JAVA で書かれていた NEM 分散台帳を C++ でリライトした catapult-client ソフトウェアを使用してデプロイされます。

プロトコルについてより詳しい情報は Technical Reference と ドキュメント を参照してください。

イメージのビルド

catapult-client をソースコードからコンパイルするには instructions に従ってください。

catapult-client の起動

catapult-client 実行可能ファイルは、様々なタイプのノードを立ち上げたり、新しいネットワークを起動したりするために使用できます。このセクションでは色々な目的で catapult-client を実行する方法について説明します。

ネットワークノードの立ち上げ

Symbol のノードを起動するには このガイド に従ってください。

開発者はテストネットノードを実験的に使用することで、貴重な資産を失うことなく、様々なトランザクションを試すこともできます。

プライベートテストネットワーク

Symbol を使用すると、企業はプロトコルレベルで custom plugins と拡張機能を開発することにより、プライベートネットワークを起動および拡張できます。Symbol Bootstrap パッケージは開発と学習目的の構築を、一つのコマンドで行うスクリプトが含まれています。

プライベートのテストネットを起動するには、このガイド に従ってください。

プライベートネットワーク

安全で本番環境に対応したプライベートチェーンを開始する方法についてはこちらをご覧ください。
